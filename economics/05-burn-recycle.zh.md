# 销毁与回收机制

在 v1 中,ATOS 供应单调不减。没有
EIP-1559 式的基础费(base fee)销毁,没有代币销毁治理动作,没有
通缩机制。本章记录当前状态,
解释其原因,并勾勒出若治理未来决定加入销毁,
这些销毁路径可能是什么样子。

## 当前状态:零销毁

追踪流经链路的每一 aatos,你会发现它去往
两个目的地之一:

```
                       Where ATOS goes
              ┌──────────────────────────────┐
              │                              │
              │   Account balance            │
              │   (spendable by holder)      │
              │                              │
              │   Module account balance     │
              │   (held by:                  │
              │     - fee_collector          │
              │     - bonded_tokens_pool     │
              │     - not_bonded_tokens_pool │
              │     - distribution_module    │
              │     - tokenomics_*_pool      │
              │     - energy_locked          │
              │     - shield contract (L2)   │
              │     - bridge escrow (L1)     │
              │   )                          │
              │                              │
              └──────────────────────────────┘
                          │
                          └─── never destroyed
```

一笔已花费的交易手续费从发送方账户进入 `fee_collector`,
再流出到验证人和社区池。ATOS 总供应量
不变。

一次燃烧押金的治理否决会销毁提案人的押金 —— 但
那些进入社区池,而非销毁。即便是被罚没的验证人
抵押也是被转移(视参数而定,进入社区池或在 bank 层
销毁),但 Atoshi 当前的 `slash_fraction_*` 都进入
distribution。

## 为何无销毁(当前理由)

三点原因:

### 1. 区块奖励是主导性的供应流

每日区块奖励发行 ≈ 10⁹ ATOS 当量(按当前测试网
参数;主网将更低)。每日交易手续费 ≈ 10⁻³ ATOS。销毁全部
手续费在发行面前几乎无感 —— 象征性通缩,实质
通胀。

对以太坊而言,EIP-1559 销毁重要是因为 gas 手续费庞大
(每年数亿美元)。对 Atoshi 而言,手续费按设计
就很小(能量系统明确对其补贴),因此手续费销毁
毫无货币政策效果。

### 2. 不应稀释验证人收入

销毁手续费会降低验证人收入。Atoshi 上的验证人已经
具有不寻常的收入机制(80% 锁定,取决于档位释放)。
在此之上再加销毁会在低档位期间恶化验证人经济,
而那恰恰是他们最需要稳定收入的时候。

### 3. 可预测的供应曲线 > 通缩表演

Atoshi 的供应曲线已经对需求内生(档位引擎)。
加入销毁会引入第二个内生变量,使供应
预测更难。投资者偏好"可解释的单调增长"而非
"带偶发通缩期的复杂动态"。

## 未来可加入哪些类型的销毁

以下每一项都是未来的治理提案,而非 v1 特性。它们
作为设计空间存在,而非承诺。

### 选项 A:固定比例手续费销毁(EIP-1559 式)

修改 `x/distribution`(或在 `fee_collector` 与验证人支付
之间插入钩子),销毁进入手续费的固定比例:

```
burn_fraction = 0.5         // governance-tunable
fee_to_burn   = collected_fees × burn_fraction
fee_to_validators = collected_fees × (1 - burn_fraction)
```

**利**:标准模式,广为人知。契合以太坊心智
模型。有意识的投资者会预期它。

**弊**:降低验证人收入。在当前的低手续费水平下,对
供应的影响可忽略。可能只有当 Atoshi 达到
以太坊主网级别活跃度时才自证其价值。

### 选项 B:动态销毁(与档位释放挂钩)

当档位引擎释放供应时,销毁一部分以对冲:

```
tier_release = circulating × release_percentage_bps / 10000
burn         = tier_release × burn_offset_fraction
net_release  = tier_release - burn
```

**利**:即便在 T3 持续期间也为净供应增长设上限。
减少分析师对"档位释放失控稀释"的担忧。

**弊**:削弱给验证人的档位释放奖励。档位引擎的
全部要义在于奖励持续的价格强度 ——
销毁会对冲掉这份奖励。

### 选项 C:隐私池退出销毁

当一笔隐私提现从 L2 的 `Shield.sol` 领取时,销毁一小
比例:

```
withdraw_amount × privacy_exit_fee_bps → burned (or sent to community pool)
```

**利**:为隐私基础设施自筹资金。可用于资助
relayer 服务或未来的 ZK 证明成本。不影响 Cosmos /
验证人经济。

**弊**:降低用户使用隐私的经济激励。隐私
本就是可选加入的;进一步征税会拖慢采用。

### 选项 D:罚没 → 销毁(而非分配)

标准 Cosmos 罚没进入社区池;在 Atoshi 上,它们可以
直接被销毁。当前参数配置为分配。

```
ParamChange: slashing_destination = "burn"
```

**利**:对作恶更强的威慑(验证人不会看到被罚没的
抵押回收进他们拥有治理投票权的社区池)。
象征性通缩。

**弊**:罚没罕见(全链每年以几十次计)。
对供应的影响可忽略。净实际变化:极小。

## 回收:"类销毁"数额目前流向何处

即便没有字面上的销毁,若干机制也通过将 ATOS
移入模块账户,把它从个人流通中移除。这些是
"回收",而非"销毁":

| 流向 | 来源 | 目的地 | 会被释放回来? |
|---|---|---|---|
| 验证人佣金 | 区块奖励 + 手续费 | 验证人账户 | 已在流通中 |
| 社区池 | 2% 手续费 + 罚没数额 | `distribution_community_pool` | 通过治理支出提案释放 |
| 能量委托抵押品 | 用户 bank 余额 | `energy_locked` 模块 | 委托到期时返还 |
| 桥 L1 托管 | 用户 bank 余额 | 桥合约持有 | L2-to-L1 提现时释放 |
| Tokenomics 锁定矿工池 | 区块奖励 | `tokenomics_miner_pool` | 通过档位引擎释放 |
| Tokenomics 锁定项目池 | 创世 | `tokenomics_project_pool` | 通过档位引擎 + 治理国库提取释放 |
| Tokenomics 未领取迁移 | 创世 | `tokenomics_migration_pool` | 截止后归入项目池 |

`circulating_supply` 查询会排除这些(见 [supply.md](./01-supply.md))。
从市值角度看,这些数额在释放前实际上是
非流通的。**把"非流通"变成"流通"的机制是
渐进且由治理控制的。**

## 对比:同类链

| 链 | 销毁机制 | 是否启用? | 年度销毁(估计) |
|---|---|---|---|
| Ethereum | EIP-1559 基础费 | 2021 年起启用 | 累计销毁约 3 M ETH |
| BNB Chain | 从交易所利润季度销毁 | 启用 | ~$1 B / year |
| Polygon | EIP-1559 销毁 | 启用 | 少量,取决于网络使用 |
| Cosmos Hub | 无(仅社区池) | 无 | 0 |
| Osmosis | 一些代币汇出口 | 微量 | < 0.1% supply / year |
| **Atoshi v1** | **无** | **无** | **0** |

Atoshi 的"无销毁"选择使其归入 Cosmos 谱系 —— 多数
BFT 链也不销毁。EVM 链(Ethereum、BNB、Polygon)倾向
于销毁,因为它们的手续费收入大到足以产生影响。

若/当 Atoshi 达到手续费收入变得可观的交易
量(全体验证人合计约 10 USD/day)时,销毁提案就
值得考虑。

## 运营说明

对于为 Atoshi 供应建模的分析师,"销毁"条目为零。
完全稀释供应量等于给定链参数与实测档位历史下的
累计发行。

对于钱包,无销毁意味着"我收到的"与"发送出去的"
之间没有不对称 —— 转账保持 ATOS 数量守恒。将
"已发送"/"已接收"数额显示为相等。

对于验证人,无销毁意味着交易手续费收入与可领取
数额 1:1 映射。经佣金 + 分配计算后,每一 aatos 手续费
最终都进入某人的账户。

## 未来考量

下一次触及货币政策的重大参数变更(可能
在主网启动或首次重大升级前后),这个问题很可能
重新打开。该提案需要处理:

- 采用哪种销毁机制(A / B / C / D,或混合)?
- `burn_fraction` 或 `burn_offset_fraction` 取多少?
- 如何向现有持有者传达供应曲线变化?
- 是否需要一次协调的链上升级(是,如果它触及
  `x/distribution` 或 `x/tokenomics`)?

链代码对此**已有准备**:`fee_collector` 流程有
可插入销毁钩子的中间步骤。加入销毁
机制是一次受控的重构,而非深度重设计。

但并不紧急。当前的零销毁均衡在内部
自洽,并契合该链稳定验证人收入 + 内生供应扩张的
设计哲学。
